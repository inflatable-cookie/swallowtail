Status: completed

## Purpose

Re-rank the next high-information provider or transport proof after DeepSeek
V4, while turning the existing exact pins into deliberate maintained version
windows where current evidence supports them.

## Generation Runway

Keep g01 active. It contains 37 numbered roadmaps and remains inside the normal
30-50 roadmap range. This checkpoint does not create g02.

## Goals

- [x] Inventory all twenty production routes and twelve common profiles by
      mechanism, access, lifecycle, topology, and compatibility posture.
- [x] Identify exact-only pins that need baseline, milestone, deprecation, or
      exclusion evidence before consumer releases can support installed-version
      ranges safely.
- [x] Revalidate current high-information candidates from official or
      maintained sources rather than inheriting the prior ranking.
- [x] Select one exact next proof or return unresolved product policy to the
      operator.
- [x] Compile only the contract, corpus, driver, and conformance work justified
      by current evidence.

## Execution Plan

- [x] Coverage, compatibility-window, and route-selection evidence: card 105.

## Candidate Pressure

The checkpoint must compare remote ACP maturity, Grok harness breadth, Z.AI
direct semantics, Cursor authority, Claude local or subscription access,
additional attached runtimes, and maintenance of existing driver windows. It
must not select a route merely to add a provider name or generalize one
provider's subscription credential into another access surface.

## Stop Condition

Stop for operator intent when equally supported choices establish product
policy, when access authority is unclear, or when a proposed route only
duplicates an existing mechanism without enough compatibility or operational
learning.

## Outcome

Research 024 audits all twenty routes and twelve profiles. Pi and DeepSeek are
the only descriptors with compatibility claims, and both are one-point
windows. Remote ACP is Draft. Grok, Claude, Cursor, Z.AI, vLLM, and SGLang
remain later or gated.

Contract 031 and roadmap 038 select attach-only Ollama native API as the first
non-singleton range proof. Stable `0.14.0` through `0.32.1` spans more than six
months with frozen intermediate qualification points. The first subset is
text-only, local unauthenticated, runtime-managed residency, and requires no
container or model acquisition.
