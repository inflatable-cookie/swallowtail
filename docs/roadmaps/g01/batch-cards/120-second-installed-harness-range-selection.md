## Objective

Revalidate the remaining installed production harnesses and select the second
compatibility-range retrofit.

## Scope

- OpenCode HTTP, Gemini CLI ACP, Kimi Code ACP, Qwen Code headless, and Pi RPC
- current stable release, Swallowtail point, release dates, and exact
  observation seam
- protocol or schema revisions, invocation and lifecycle milestones, retained
  state, configuration posture, and known exclusions
- current official provider or maintained-project documentation, release
  records, tagged source, schemas, and exact package artifacts
- compare range information gained against implementation cost
- promote one dated research delta and compile cards 121-123 around the winner
- no production driver change, live credential, installation, or consumer edit

## Acceptance Criteria

- [ ] every candidate has a current exact compatibility posture
- [ ] observation authority is separate from protocol and executable versions
- [ ] the recommended range has exact corpus and rejection points
- [ ] no behavior transition is crossed by inference
- [ ] support authority and deprecated posture remain explicit
- [ ] implementation scope is materially different from the Codex proof
- [ ] one sole next task remains or an operator decision is requested
