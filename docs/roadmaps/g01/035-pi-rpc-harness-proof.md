## Bounded First Proof

One local or remote-authoritative host launches exact Pi `0.80.10` in RPC and
ephemeral-session mode. The immutable plan binds an operator-selected
downstream provider, model route, model, delegated harness authentication,
working directory, and `AmbientHost` posture.

The route enables `read`, `grep`, `find`, and `ls` as harness-owned read-intent
tools. It disables write, edit, bash, custom tools, extensions, skills, prompt
templates, context files, package work, update checks, telemetry, and automatic
retry. These restrictions limit behavior; they do not claim containment.

The session permits one active model operation, two completed prompts, one
steering message, and one follow-up. Extension dialogs relay through correlated
callbacks. Display-only UI requests become bounded observations. Native abort,
host cancellation, deadline, process supervision, and joined cleanup remain
independent.

## Stop Condition

Stop if Pi cannot bind provider and model explicitly, RPC requires ambient
customization or retry, the first useful subset needs write or shell authority,
command responses cannot be separated from model completion, callback payloads
cannot stay bounded and redacted, or deterministic fixtures require a Pi
install, credential, paid request, browser, or external network.
