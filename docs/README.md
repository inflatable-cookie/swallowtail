# Swallowtail Documentation

This tree is Swallowtail's authority surface. Consumer repositories may record
their own adoption plans, but Swallowtail's portable contracts and roadmap live
here.

## Authority Spine

- [Vision](vision/README.md) — product intent and boundaries
- [Architecture](architecture/README.md) — realized structure and guardrails
- [Contracts](contracts/README.md) — durable rules and public boundaries
- [Specs](specs/README.md) — provisional planning and unresolved decisions
- [Roadmaps](roadmaps/README.md) — sequenced delivery generations
- [Logs](logs/README.md) — decisions and validation evidence
- [Research](research/README.md) — evidence awaiting promotion
- [Policy](policy/internal-writing-style.md) — repository working style

## Authority Rule

When sources disagree:

1. active contracts govern behavior
2. realized architecture governs current structure
3. vision governs direction
4. the active roadmap governs sequence
5. specs and research remain provisional

Nucleus and Soundcheck are initial migration sources. Monkey may later provide
a local-model route. None is an authority owner for Swallowtail.
